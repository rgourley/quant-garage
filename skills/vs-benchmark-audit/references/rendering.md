# Rendering: vs-benchmark-audit

Note-mode. Layout:

1. Header (2 lines: identity + verdict)
2. Return statistics block (compressed tearsheet)
3. vs-benchmark block (beta, alpha, tracking error, rolling IC)
4. Take + caveats

Verdict tags:
- `REAL ALPHA (deflated Sharpe significant)`
- `POSSIBLY ALPHA (positive but not DSR-significant)`
- `ESSENTIALLY BETA (returns explained by benchmark)`
- `UNDERPERFORMING`
- `NO EDGE EVIDENT`
